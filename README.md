> NetCop — инструмент администрирования сети для малых организаций (учебные классы, коворкинги, локальные офисы). Позволяет распределять полосу пропускания между процессами и контролировать сетевую нагрузку на рабочих станциях.

# NetCop

NetCop — это мощная утилита для мониторинга сети и шейпинга трафика (ограничения скорости) на уровне процессов для Windows. Состоит из фонового агента-службы на управляемых рабочих станциях и центрального сервера с веб-дашбордом для управления.

## Основные возможности

- **Детальный мониторинг:** Телеметрия процессов (CPU, RAM, активные соединения) и отображение топ-10 процессов, потребляющих трафик.
- **Traffic Shaping (IN+OUT):**
  - Ограничение **исходящего** трафика через политики `PowerShell NetQosPolicy`.
  - Ограничение **входящего** трафика через драйвер `WinDivert` и кастомный алгоритм Token Bucket (дроп пакетов для естественной отработки TCP).
- **Групповые политики:** Группировка процессов по категориям (игры, торренты, браузеры) для применения лимитов в один клик. Имеется глобальный **Priority Mode**.
- **Service mode:** агент работает как фоновая служба с индикатором в системном трее Windows.
- **Персистентность:** Все установленные лимиты и логи аудита надежно сохраняются в локальной SQLite базе данных сервера.

## Архитектура

- **Сервер (`server/`):** FastAPI, SQLite (для хранения лимитов и аудита), веб-дашборд на чистом HTML/JS/CSS (Chart.js). Авто-генерация ключа и конфигурации.
- **Агент (`agent/`):** Python-скрипт (`psutil`, `pydivert`, `pystray`), опрашивающий ОС и получающий команды от сервера через long-polling.

## Требования

- Windows 10/11 (Агент работает только на Windows).
- Python 3.8+ (для запуска из исходников).
- **Права Администратора** для запуска Агента (необходимы для `WinDivert` и политик QoS).

## First run (Первый запуск)

1. Запустите сервер: `python launcher.py --host 0.0.0.0`
2. Сервер напечатает сгенерированный API-ключ — скопируйте его.
3. На рабочей станции запустите `install.bat` от имени Администратора.
4. Введите адрес сервера и скопированный ключ.
5. Откройте дашборд в браузере по адресу из консоли сервера.

## Сборка в `.exe` (PyInstaller)

Проект полностью готов к сборке в единые бинарные файлы:

- **Сборка сервера:** Запустите `server/build_server.bat`. Готовый файл появится в `server/dist/NetCopServer.exe`.
- **Сборка агента:** Запустите `agent/build_agent.bat`. Файл появится в `agent/dist/NetCopAgent.exe`. 
*(Внимание: если вы используете WinDivert, вам потребуется положить файлы драйвера `WinDivert.dll` и `WinDivert64.sys` рядом с exe-файлом агента, либо добавить их при сборке через `--add-binary`).*

## Безопасность

Связь между агентами и сервером (а также доступ к API) защищена с помощью общего секретного ключа (`NETCOP_API_KEY`), который передается в заголовке `X-NetCop-Key`. Убедитесь, что ключ совпадает на клиенте и сервере.
