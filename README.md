# NetCop

NetCop — утилита для мониторинга сети и **шейпинга трафика на уровне процессов** под Windows. Скрытый агент на целевой машине следит за расходом полосы пропускания, замедляя жадные приложения.

Главное отличие от наивных «ограничителей»: NetCop умеет честно резать **скачивание**, а не только отдачу.

## Почему download-шейпинг — это сложно (и как тут решено)

Большинство самодельных шейперов пытаются ограничить скачивание, **дропая входящие пакеты**. Это не работает: они уже попали на WAN-сегмент, занимая канал. NetCop применяет более инженерный подход.

NetCop использует три слоя, потому что одним трюком всё не покрыть:

| Слой                    | Что шейпит | Механизм                 |
|-------------------------|-----------|--------------------------|
| **TCP-загрузка** (HTTP, лаунчеры, обновления) | download | **TCP Window Clamping** |
| **UDP / QUIC / uTP** (часть игр, ютуб, торрент-uTP) | download | Delay-очередь + token bucket |
| **Отдача** (любая)               | upload   | `NetQosPolicy` (Windows QoS) |

### TCP Window Clamping — сердце проекта

TCP-отправитель физически не может слать больше, чем рекламируемое получателем окно. Ваша машина — получатель, и NetCop ограничивает это окно по формуле:

```
target_window = target_rate × RTT
```

Отправитель сам себя тормозит. **Байт не выходит на провод вообще** — ноль WAN-оверхеда, ноль потерь, ноль ретрансмиссий.

Window scale (множитель окна) считывается из TCP-хендшейка и учитывается при пересчёте.

### Честные ограничения

- **Window clamping применяется только к TCP.** QUIC/UDP/uTP ведут своё congestion control — для них работает delay-очередь (придержать пакеты).
- **RTT задаётся в конфиге** (по умолчанию 50 мс). Для задачи «не дать процессу сожрать весь канал» точность до миллисекунд не нужна.
- Если агент не застал TCP-хендшейк (соединение уже было открыто), для него используется scale = 0 — окно будет чуть меньше.
- **Это endpoint-шейпинг.** Идеальный шейпинг инбаунда живёт на роутере; здесь его нет, поэтому TCP закрывается клампом.

## Возможности

- **Мониторинг:** CPU, RAM, активные соединения, топ-10 процессов по трафику.
- **Шейпинг IN+OUT:** кламп окна для скачивания (TCP) + QoS для отдачи + delay-пейсинг для UDP.
- **Массовое управление:** процессы группируются по категориям (торренты, игры, стриминг, браузеры) — лимит для категории изменяется одной командой.
- **Priority Mode:** автоматически придушивает «жручие» категории, запоминает прежние лимиты и **восстанавливает их** при отключении режима.
- **Network Kill с авто-восстановлением:** вырубить сеть машины можно одной кнопкой, но это **деретёр, а не кирпич** — сеть восстанавливается сама.
- **Аудит-лог:** все действия пишутся в SQLite с временем, целью и source IP.
- **System Tray:** агент может жить фоном с индикатором в трее.

## Архитектура

- **Сервер (`server/`):** FastAPI + SQLite, веб-дашборд (HTML/JS/Chart.js).
  - Команды доставляются с гарантией **at-least-once**: висят в очереди, пока агент их не подтвердит (ACK). Потерянный ответ гарантированно дойдёт до получателя.
  - Запись в SQLite вынесена в отдельный поток — хендлеры не блокируют event loop на диске.
  - Сравнение API-ключа — constant-time (`secrets.compare_digest`).
- **Агент (`agent/`):** Python (`psutil`, `pydivert`, `pystray`). Репортит метрики, тянет и **подтверждает** команды, держит watchdog на kill.

## Требования

- Windows 10/11 (агент только под Windows).
- Python 3.8+ (для запуска из исходников).
- **Права администратора** для агента (нужны для WinDivert и QoS).

## Установка и запуск

### Сервер

1. `cd server`
2. `pip install -r requirements.txt`
3. `python launcher.py`
   - При первом запуске сгенерируется API-ключ и напечатается в консоль крупным блоком. **Сохраните его** — он понадобится агентам.
4. Дашборд: `http://127.0.0.1:8000`

Bind-адрес и порт настраиваются через `server/config.json` или аргументами:
```
python launcher.py --host 0.0.0.0 --port 8000
```
По умолчанию слушает только `127.0.0.1`. Чтобы агенты с других машин могли подключаться — поставьте `--host 0.0.0.0` и пропишите портфорвардинг.

### Агент

1. `cd agent`
2. Запустите `install.bat` (от администратора). Скрипт спросит:
   - URL сервера
   - **API-ключ (обязательно)** — тот, что напечатал сервер
   - Включить ли WinDivert-шейпер (и RTT для клампа)
   - Запускать ли в трее
3. Агент стартует автоматически. **Только от имени администратора.**

## Сборка в `.exe` (PyInstaller)

- **Сервер:** `server/build_server.bat` → `server/dist/NetCopServer.exe`
- **Агент:** `agent/build_agent.bat` → `agent/dist/NetCopAgent.exe`

> При использовании WinDivert положите `WinDivert.dll` и `WinDivert64.sys` рядом с exe агента (или добавьте через `--add-binary`). Драйвер скопируется автоматически.

## Безопасность

- Связь агент↔сервер защищена общим секретом в заголовке `X-NetCop-Key`.
- **Важно:** ключ ходит **плейнтекстом**. В пределах LAN это приемлемо, но в недоверенной сети поднимите сервер за TLS.
- Дефолтный ключ "secret" и ключи короче 16 символов отклоняются — сервер не стартует.

## Что изменилось в этой версии

Переписано после ревью. Основное:

1. **Download-шейпинг теперь настоящий** — добавлен TCP window clamping. Старый код дропал входящие пакеты (loss-throttle с двойной загрузкой).
2. **Token bucket больше не дедлочится** на низких лимитах. Был `burst = rate × 0.5` → ниже ~24 кбит/с burst падал под один MTU и `congestion_window` блокировался.
3. **Команды не теряются.** Очередь чистится только после ACK от агента, а не сразу на выдаче. Потерянный ответ гарантированно дойдёт.
4. **Двойное списание токенов убрано.** Per-process и global бакеты теперь координируются через peek/commit — пакет либо списывается, либо отбрасывается, но дважды не трогается.
5. **Network Kill безопасен.** Добавлены unkill + watchdog авто-восстановления. Раньше kill был билетом в один конец — только перезагрузка сети помогала.
6. **Безопасность:** constant-time сравнение ключа, исправлена невалидная пара `CORS allow_origins=["*"] + allow_credentials=True`.
7. **Прочее:** определение интерфейса по дефолтному маршруту (а не первому "up", которым мог оказаться виртуальный адаптер).
